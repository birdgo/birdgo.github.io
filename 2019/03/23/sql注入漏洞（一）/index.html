sql注入漏洞（一）

Mar 23, 2019

漏洞原理、危害、防护

1、漏洞原理
SQL注入漏洞主要形成原因是在网站的数据交互中，前端的数据传入到后台处理时，没有做严格的判断，导致其传入的”数据”拼接到SQL语句中后，被当作SQL语句的一部分执行，使用户可以精心构造一些语句进行越权操作，从而导致数据库受损。

2、漏洞危害
一：可直接造成数据库里的信息泄露，如员工或管理员账号密码信息；
二：操作数据库，例如增加或删除账号，写入一句话木马等。

3、漏洞防护
一：参数化查询，把输入参数作为一个值执行，不会当作SQL语句执行
二：对带入SQL语句中的外部参数进行转义或过滤
三：限制用户对数据库的操作权限
四：关闭数据库错误信息显示
五：使用waf
